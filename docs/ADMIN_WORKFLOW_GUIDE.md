# Admin Workflow Guide

The browser UI exposes the core internal workflow:

1. Select country and city.
2. Review city viability and why population-only logic is insufficient.
3. Inspect generated territories and their validation states.
4. Use the rerun control to re-run analysis for the selected city.
5. Download contract-ready exports for approved packs.

The sandbox build uses system-driven review states. Production should connect these states to authenticated reviewer actions and persistent audit logs.
